# Reveal Item Experiment

## Goal

Prototype a charge-based reveal pulse that can be evaluated in the noise theme without committing it to the main game design.

## Status

This document exists only for the `experiment/reveal-item-pulse` branch.
The feature is experimental and may be discarded entirely.

## Evaluation Scope

The experiment should answer only these questions:

- Does an earned reveal pulse improve the game?
- Does it preserve enough of the motion-driven identity?
- Does it feel better than manual shake and auto shake alone?

If the answer is no, discard the branch.

## Constraints

- Item accrual and manual item use are always active in this branch.
- Normal-mode UI remains as close to the main branch as possible aside from the added item display and auto-use control.
- The feature must work in both themes at the presentation layer, even if the practical value is mainly in the noise theme.
- The implementation should be minimal and disposable.
- Do not redesign scoring, progression, or core controls beyond what is needed to test the feature.

## Minimal Prototype

### Gameplay state

Add only the minimum state needed:

- `revealCharges`
- `piecesTowardNextRevealCharge`

### Accrual rule

- Gain 1 charge every 10 locked pieces.
- Carry piece remainder toward the next charge.
- Cap charges at 3.

### Use rule

- The player may spend 1 charge for an instant reveal pulse.
- The pulse should not pause gameplay.
- If no charge is available, the action does nothing.

### Input/UI

- Add a sidebar `AUTO USE` toggle.
- Add a sidebar display with:
  - a 10-step gauge for progress toward the next charge
  - repeated item markers instead of a numeric charge count
- Use `D` for manual item use in this branch.
- Preserve `AUTO USE` across `R` restarts and reloads.
- If `AUTO USE` is turned on while charges are available, spend 1 immediately.
- After that, whenever a new charge is gained, spend 1 automatically.

### Presentation rule

On use, apply a long dark reveal pulse to the same gameplay-relevant rendered roles that participate in shake:

- active piece
- settled stack
- frame / wall
- line-clear overlay
- preview pieces

The pulse should:

- darken those roles substantially at the start
- fade slowly over roughly 2 to 3 seconds
- not switch themes
- not change gameplay rules

### Sound

Reuse existing sound paths if helpful, but do not add new audio assets just for the experiment.

## Non-goals

- No new item inventory system beyond reveal charges.
- No multiple item types.
- No changes to normal-mode UX.
- No permanent addition to the public design doc unless the experiment succeeds.

## Merge Criteria

Promote the feature into the main design only if:

- it materially improves playability
- it does not make the game feel like ordinary falling-block play with a visibility power-up
- it still feels architecturally clean

Otherwise discard the branch.
